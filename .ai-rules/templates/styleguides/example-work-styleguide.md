# Example Work Styleguide

This is a public example styleguide for professional/business writing
contexts. Replace it with your own private styleguide if desired.

## General Style

- Be clear and respectful.
- Lead with the conclusion or ask.
- Keep paragraphs short.
- Cite specifics over generalities (numbers, dates, file paths, names).
- Default to plain language; jargon only when it carries real meaning.

## Tone

- Professional without being stiff.
- Confident without being grandiose.
- Honest about uncertainty — flag what you don't know.

## Avoid

- Hyperbole ("game-changing", "revolutionary")
- Filler intros ("I hope this finds you well")
- Buzzwords without substance ("synergy", "leverage", "ecosystem" used
  loosely)
- Walls of text — break up dense paragraphs with structure

## Format

- Subject lines and PR titles: Title Cased and informative
- Bulleted lists for parallel items, numbered lists for sequenced steps
- Code, file paths, and commands in inline code formatting
